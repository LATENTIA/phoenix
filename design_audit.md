# Phoenix — Design & UI Audit

Post-redesign audit of the Phoenix UI. Reviewed against the locked decisions in the original redesign proposal and the current codebase (`dashboard.html`, `tokens.css`, partials, `dashboard.js`, `share_dashboard.html`, report CSS/JS, `app.py`).

**Audit date:** June 2026  
**Scope:** Visual design, layout architecture, UX, and UI-adjacent security/performance notes.

## What shipped well

### Night ledger design system

- **`tokens.css`** — single source of truth for colors, fonts, radii
- **Phoenix green** (`#16a34a`) aligned with logo; cyan removed as global accent
- **Business accounts** — warm gold `#d4a012` (not neon amber)
- **`--tax-due`** (`#7dd3fc`) reserved for “amount owed” figures
- **Typography** — Newsreader (headings), Source Sans 3 (UI), IBM Plex Mono (numbers)
- **Legacy aliases** (`--panel`, `--accent`, etc.) kept for gradual migration

### Shell layout (`dashboard.html` + `dashboard.css`)

- **Sidebar** with Filing nav + Data actions at bottom
- **Sync from IBKR** — primary green button, no emoji
- **Add trade** — secondary ghost button
- **Update basis prices** — hidden unless CGT tab active (`dashboard.js`)
- **Account dropdown** (`<details>`) replaces pill switcher + uppercase badge
- **Sync status** — human line: “631 trades ingested · last sync …”
- **Debug counts** — collapsed in `<details class="tech-details">`
- **Responsive** — sidebar collapses to horizontal strip below 900px
- **Single privacy toggle** in top bar; persisted via `localStorage`

### No iframe (main dashboard)

- Reports fetched as **`?partial=1`** HTML fragments
- **`templates/partials/*_content.html`** for all major reports
- **`.report-shell`** wrapper scopes report CSS away from shell chrome
- **`templating.render_report(as_partial=True)`** — dual mode with standalone export
- **Server-side first paint** — `initial_report_html` inlined on `/` when data exists
- **Script re-execution** after `innerHTML` inject (tab switches)
- Report CSS linked once in dashboard `<head>` (tob, pnl, cgt, dividends)

### TOB report improvements

- **Buy/Sell** derived from quantity sign when missing from DB
- **Crypto quantities** — fractional formatting (no truncated `0` BTC)
- **Filing view / Full detail** toggle with `data-col` column hiding
- Empty “Generated by IBKR” line hidden when no timestamp

### Security fixes (UI-adjacent)

- **Share token log redaction** — `_safe_path()` → `/share/abc12345…REDACTED`
- **Auth exposure warning** — `_warn_if_auth_disabled_and_exposed()` on non-loopback bind

## What’s still wrong

### P1 — Visible gaps

#### 1. Share dashboard not migrated

`share_dashboard.html` still uses:

- Legacy horizontal **tabs** (not sidebar)
- **Iframe** + `embed=1` + cache-bust timestamp
- Inline style overrides to fight `dashboard.css` grid rules
- No `dashboard.js` partial boot path

Accountants on share links get the old two-app experience.

**Fix:** Reuse sidebar shell (read-only); fetch `?partial=1` like main dashboard; drop iframe.

#### 2. Empty state off-brand

`empty_report.html` still uses old slate (`#0f172a`), system font, and `⚡ Load data` copy.

**Fix:** Use `tokens.css`; copy “Sync from IBKR”; match night ledger empty placeholder in `#viewer`.

#### 3. Hero metrics not implemented

Design called for one **hero number** per report (e.g. “€98.59 TOB due”). Reports still open with **5–9 identical KPI cards** and uppercase micro-labels.

**Affected:** TOB, CGT, Dividends, Corporate tax, P&L.

**Fix:** Hero strip in shell (fed by report JS via `postMessage`) or replace card grid with one large mono figure + subtitle row.

#### 4. Report headers redundant

Partials still include full `header` + `h1` + “Report built: …” inside `#viewer`. Shell already shows account context in top bar.

**Fix:** Slim partial header (period + filters only) or context line under sidebar active item.

---

### P3 — Interaction & performance

| Issue | Behavior | Fix |
|-------|----------|-----|
| Full page reload after sync | `loadData()` → `location.reload()` | Update sync status + refresh active partial only |
| Tab switch always refetches | `showReport()` → fetch every click | Client cache per `(account, report)` |
| “Loading…” flash | Optimistic placeholder on every tab | Skip fetch if cached partial fresh |
| `match_lots()` uncached | Server recomputes on every `?partial=1` | Ingest-time cache or LRU keyed on last ingest |
| Performance routing | Maps to `pnl?tab=performance` | Acceptable per locked decision; could get own partial later |

## Bottom line

The main Phoenix app successfully moved from “Claude standards dashboard” to a **coherent night-ledger product** with sidebar navigation, brand-aligned green, and a proper partial-based report shell. The redesign is **~75% complete** on the primary user path.

The largest remaining design debt is **share dashboard parity**, **hero metrics**, and **empty/settings pages** still on the old visual language. Addressing share + hero strip would close the gap between “redesigned” and “finished.”
